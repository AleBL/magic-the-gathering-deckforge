### **Example: Adding a Card to Deck**

1. User clicks "Add to Deck" in CardDetailsModal
2. `useDeckActions.handleAddToDeck(card)` is invoked

4. `useDeckManager.saveDeck()` writes to Dexie
5. `useDeckTokens` auto-populates related tokens
6. `useToast` shows success message

## State Management: Zustand

**File**: `src/store/useDeckStore.ts`

**Global State Shape**:
```typescript
DeckStoreState {
  currentDeck: Card[]                          // Cards in workspace
  currentDeckRelatedTokens: DeckRelatedToken[] // Auto-populated tokens
  editingDeck: EditingDeckState {
    deckId: string | null
    deckName: string
    deckFormat: DeckFormat                     // Standard, Modern, Commander, etc.
    deckNotes?: string
  }
  pendingAction: string | null                 // 'save-deck' | 'playtest-deck' | 'clear-deck'
}
```

**Key Mutations**:
- `addCard(card)` â†’ Add card to currentDeck
- `removeCard(cardId)` â†’ Remove and auto-cleanup related tokens
- `updateCard(updatedCard)` â†’ Swap printing, foiling, etc.
- `toggleCommander(cardId)` â†’ Mark/unmark commander (enforces 1 or 2 max)
- `loadDeckToEdit(id, name, format, cards, notes, tokens)` â†’ Populate edit state
- `clearDeck()` â†’ Reset all state

**Why Zustand?**:
- Minimal boilerplate, no providers needed in most cases
- Immutable updates prevent accidental mutations
- Fine-grained subscriptions (components only re-render on relevant changes)
- Supports both React and plain JS

## Persistence: Dexie (IndexedDB)

**File**: `src/db/database.ts`

**Schema**:
```javascript
// Decks table
id (primary key)
name
format (Standard, Modern, Commander, etc.)
cards (Card[] serialized as JSON)
tokens (DeckRelatedToken[] serialized as JSON)
notes
createdAt
updatedAt

// Cards table (optional, for caching Scryfall data)
scryfall_id (primary key)
name
set
foil
printing_index
```

**Usage**:
- `useDeckManager` loads/saves decks to IndexedDB
- Full offline support: once a deck is saved, it's available without internet
- Auto-migrations handled by Dexie version bumps

**Why Dexie?**:
- Thin wrapper over IndexedDB API
- Promises-based (no callbacks)
- Type-safe queries
- React hooks support via `dexie-react-hooks`

## External Integrations

### **Scryfall API** (via `scryfall-sdk`)

**Integration Points**:
- `useCardSearch` â†’ `/cards/search` endpoint (debounced)
- `useCardPrints` â†’ Get alternative printings for a card
- `useCardRelatedTokens` â†’ Find tokens generated by card effects
- Legality checks â†’ Query format restrictions (Standard, Modern, Commander, etc.)

**Error Handling**:
- Debounced requests to avoid rate limiting
- Fallback UI when API is down
- Cached results in Zustand for quick re-renders

### **File Import/Export**

**Services**:
- `deckImportService.ts` â†’ Parse MTG Arena format (`.txt`), `.DEC` files
- `fileDownload.ts` â†’ Export to JSON, `.DEC`, proxy sheets

**Supported Formats**:
- MTG Arena deck format (card name + quantity)
- Cockatrice `.DEC` format
- JSON (custom Deck schema)

## Deployment

### **Electron (Desktop)**
- Built with `electron-builder`
- Targets: Windows (NSIS + zip), macOS (DMG + zip), Linux (AppImage + deb + zip)
- Build: `yarn build:win` / `yarn build:mac` / `yarn build:linux`

### **Web (Browser)**
- Single-page app (SPA) served by Vite
- PWA support via vite-plugin-pwa
- Build: `yarn build:vite`

**Single Codebase**:
- Shares all hooks, components, and state
- Separate entry points: `src/main.tsx` (web/Electron), `electron/index.ts` (main process)
- No code duplication between targets

## Common Patterns

### **Optimistic Updates**
Components update UI immediately, then save in background:
```typescript
store.addCard(card);           // Instant UI update
manager.saveDeck();            // Async DB write
```

### **Error Boundaries**
`src/components/ErrorBoundary.tsx` catches render errors and displays user-friendly fallback UI.

### **Hooks for Reuse**
Custom hooks encapsulate complex logic (Scryfall API, deck validation, playtest rules). Components remain simple and declarative.

### **Translations**
Every user-facing string is wrapped in `t()` for multi-language support.
